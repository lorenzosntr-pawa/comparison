# Phase 110.1-01-BUG035-FIX: Competitor-Only Events Missing Odds

**Objective:** Fix BUG-035 where competitor-only events (no BetPawa match) show no odds on the Competitors-Only tab after v2.9 migration.

## Problem

The v2.9 async write path skips competitor-only events because:
1. `market_odds_current` is keyed by BetPawa event_id
2. Competitor-only events have no BetPawa event_id
3. Code explicitly skips these at lines 1732-1736 and 1680

## Solution

Add legacy table writes for competitor-only events in the async path. The read path (`_load_latest_competitor_snapshots`) already queries legacy tables, so no read path changes needed.

## Tasks

### 1. Modify async path to write competitor-only events to legacy tables

**File:** `src/scraping/event_coordinator.py`
**Location:** After line 1769 (after market write batch enqueue)

Add logic to:
- Collect competitor-only events (where `betpawa_eid is None`)
- Create `CompetitorOddsSnapshot` + `CompetitorMarketOdds` ORM objects
- Write to database (similar to sync path at lines 1799-1826)

```python
# After line 1769: Handle competitor-only events (no BetPawa match)
# These need legacy table writes since market_odds_current requires event_id
comp_only_snapshots: list[tuple[CompetitorOddsSnapshot, list[CompetitorMarketOdds]]] = []

for idx, cswd in enumerate(comp_write_data):
    sr_id_val, source_value, comp_event_id = comp_meta[idx]
    betpawa_eid = event_id_map.get(sr_id_val)

    if betpawa_eid is None:
        # Competitor-only event: write to legacy tables
        orm_markets = [
            CompetitorMarketOdds(
                betpawa_market_id=m.betpawa_market_id,
                betpawa_market_name=m.betpawa_market_name,
                line=m.line,
                handicap_type=m.handicap_type,
                handicap_home=m.handicap_home,
                handicap_away=m.handicap_away,
                outcomes=m.outcomes,
                market_groups=m.market_groups,
                unavailable_at=m.unavailable_at,
            )
            for m in cswd.markets
        ]
        snapshot = CompetitorOddsSnapshot(
            competitor_event_id=comp_event_id,
            scrape_run_id=scrape_run_id,
        )
        comp_only_snapshots.append((snapshot, orm_markets))

# Write competitor-only snapshots to legacy tables
if comp_only_snapshots:
    for snapshot, _ in comp_only_snapshots:
        db.add(snapshot)
    await db.flush()  # Get snapshot IDs

    for snapshot, markets in comp_only_snapshots:
        for market in markets:
            market.snapshot_id = snapshot.id
            db.add(market)

    await db.commit()
    logger.info(
        "Wrote competitor-only snapshots to legacy tables",
        count=len(comp_only_snapshots),
    )
```

## Considerations

- **No cache update for competitor-only events:** The cache is keyed by BetPawa event_id. Competitor-only events will serve from database. This is acceptable since:
  - Competitor-only tab is less frequently accessed
  - Cache hit rate for competitor-only is already 0%
  - Adding a separate cache key scheme adds complexity

- **No change detection for competitor-only:** Each scrape will write new snapshots. This matches pre-v2.9 behavior. Storage impact is minimal since competitor-only events are rare.
