<task type="auto">
  <name>Task 1: Run static analysis to identify dead code</name>
  <files>web/src/**/*.ts, web/src/**/*.tsx (read-only scan)</files>
  <action>
Use multiple approaches to identify dead code:

1. **Scan for unused imports** using grep/analysis:
   - Find imports at top of files
   - Check if imported names are used in the file
   - List candidates for removal

2. **Identify unused exports** by tracing usage:
   - Scan all exported functions/components/types
   - Search for usage across the codebase
   - Focus on likely dead code areas:
     - Any SSE-related code that may remain (useSSE*, SSE types)
     - Hooks that may be unused after migrations
     - Components not imported anywhere
     - Type definitions not referenced

3. **Check for deprecated patterns:**
   - Search for "TODO", "FIXME", "DEPRECATED" comments
   - Search for SSE-related code (`EventSource`, SSE endpoint patterns)
   - Search for commented-out code blocks

4. **Check barrel exports (index.ts files):**
   - Verify all re-exports are actually used
   - Flag exports with no external consumers

**Output:** Create a list of dead code candidates categorized by:
- HIGH CONFIDENCE: Clearly unused (no consumers found, not an entry point)
- MEDIUM CONFIDENCE: Potentially unused (internal use only, may be needed)
- LOW CONFIDENCE: Needs verification (might be used dynamically)

Do NOT delete anything in this task - only analyze and document findings.
  </action>
  <verify>Analysis complete with categorized findings documented</verify>
  <done>Dead code candidates identified and categorized by confidence level</done>
</task>

<task type="auto">
  <name>Task 2: Remove confirmed dead code</name>
  <files>Files identified in Task 1</files>
  <action>
Based on Task 1 findings, remove confirmed dead code:

1. **Remove unused imports** (HIGH CONFIDENCE only):
   - Delete import lines that are clearly unused
   - Preserve type-only imports used for documentation

2. **Remove unused exports/components/hooks** (HIGH CONFIDENCE only):
   - Delete function/component definitions with no consumers
   - Update barrel exports (index.ts) to remove dead re-exports
   - For entirely unused files: Delete the whole file

3. **Clean up deprecated patterns:**
   - Remove TODO/FIXME comments that are no longer relevant
   - Remove any remaining SSE-related code
   - Remove commented-out code blocks

**Preserve:**
- Entry points (App.tsx, main.tsx, routes.tsx)
- shadcn/ui primitives in components/ui/ (may be used later)
- Type definitions that provide documentation value
- Components used in routes even if not directly imported elsewhere

**Do NOT remove MEDIUM or LOW confidence items** - document them for future review.
