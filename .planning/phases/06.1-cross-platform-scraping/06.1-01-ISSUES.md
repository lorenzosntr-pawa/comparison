## Open Issues

### UAT-003: Bet9ja scraping fails - external API unreachable

**Discovered:** 2026-01-21
**Phase/Plan:** 06.1-01
**Severity:** Major
**Feature:** Bet9ja scraping
**Description:** Bet9ja scraping reports `success: false` with an empty error message. Investigation shows the Bet9ja API (sports.bet9ja.com) is completely unreachable from the current environment (HTTP 000 - connection timeout).
**Expected:** Bet9ja events should be discovered via tournament browsing
**Actual:** API unreachable, empty error message due to timeout exception
**Repro:**
1. Trigger a scrape via POST /scrape
2. Observe Bet9ja returns success: false, error_message: ""
**Root cause:** External API availability issue - not a code bug. May be:
- Geo-blocking (requires Nigerian IP)
- Temporary downtime
- Network/firewall restrictions
**Status:** DEFERRED - Monitor API availability or consider VPN/proxy options

## Resolved Issues

### UAT-001: BetPawa competition parsing fails - wrong response key

**Discovered:** 2026-01-21
**Resolved:** 2026-01-21 - Fixed in 06.1-01-FIX.md
**Commit:** bf04d5d
**Severity:** Blocker
**Feature:** BetPawa scraping
**Description:** The orchestrator failed to extract competition IDs from BetPawa API response. The code looked for `regions` key at the top level, but the API returns data under `withRegions[0].regions`.
**Fix:** Changed `categories_data.get("regions", [])` to `categories_data.get("withRegions", [])[0].get("regions", [])`

### UAT-002: SportyBet health check endpoint returns 404

**Resolved:** 2026-01-21 - Fixed in 06.1-01-FIX.md
**Commit:** 1aa882c
